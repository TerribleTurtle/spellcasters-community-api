# v0.2.0 Schema Changes

**Date:** 2026-03-07 · **Breaking:** None

## `match_xp.summon_xp` (Required)

New object in `game_systems.match_xp`. XP granted on unit placement.

```json
"summon_xp": { "rank_i": 50, "rank_ii": 150, "rank_iii": 300, "rank_iv": 500 }
```

## `map_objects` (Optional)

New top-level object in `game_systems.json` for persistent map structures.

```json
"map_objects": {
  "lifestone": { "heal_per_sec": 10, "heal_target": "Spellcaster", "heal_range": "territory" }
}
```

Guard with a null check — this field may be absent in older responses.

## `starting_knowledge.early_access_compensation` (Required)

New field in `game_systems.progression.starting_knowledge`. One-time 2,000 Knowledge grant for all players active before March 2, 2026.

```json
"starting_knowledge": { "default": 250, "beta": 1000, "early_access_compensation": 2000 }
```

## `spawner[].max_active` (Optional)

New optional field on spawner entries. Maximum number of spawned units alive simultaneously.

```json
{ "unit_id": "harpy", "count": 3, "interval": 10, "max_active": 12, "trigger": "Interval" }
```

If absent, assume no cap. All buildings now use `count: 3` (was 5), `max_active: 12` (was uncapped/15), and death spawn `count: 6` (was 10).
